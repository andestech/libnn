void riscv_nn_activate_s8 ( q7_t *  in_out,
uint32_t  size,
uint16_t  int_bits,
riscv_nn_activation_fun  act_fun 
)

This function performs activation on signed 8-bit integer input vectors using either the Sigmoid or Tanh function, along with in-place and look-up table (LUT) algorithms.

Parameters
[in,out]in_outPointer to the input/output vector
[in]sizeNumber of elements in the input/output vector
[in]int_bitsNumber of the bits in the integer part should be less than 4
[in]act_funSelection of activation functions. See Note below for details.
Returns
None
Note
The available activation functions for selection include:
  • NN_SIGMOID: Use the Sigmoid activation function
  • NN_TANH: Use the Tanh activation function

Example:

#define SIZE 32
q7_t in_out[SIZE] = {...};
riscv_nn_activate_s8(in_out, SIZE, 0, NN_SIGMOID);

◆ riscv_nn_activate_s16_hp()

void riscv_nn_activate_s16_hp ( const q15_t *  in_vec,
q15_t *  out_vec,
const uint32_t  size,
const uint32_t  left_shift,
const riscv_nn_activation_fun  act_fun 
)

This function performs activation on signed 16-bit integer input vectors using either the Sigmoid or Tanh function, along with a high-precision algorithm.

Parameters
[in]in_vecPointer to the input vector
[out]out_vecPointer to the output vector
[in]sizeNumber of elements in the input/output vector
[in]left_shiftThe scaling for the inputs. See Note below for details.
[in]act_funSelection of activation functions. See Note below for details.
Returns
None
Note
  • Let INT_BITS represent the number of the bits in the integer part of the inputs. Set left_shift to INT_BITS -3 if INT_BITS is 3 or greater. Otherwise, set left_shift to 0.
  • The inputs should be scaled down by the following if INT_BITS is less than 3: 1/(2^(3 - INT_BITS))
  • The availbale activation functions for selection include:
    • NN_SIGMOID: Use the Sigmoid activation function
    • NN_TANH: Use the tanh Activation function

◆ riscv_nn_leaky_relu_s8()

void riscv_nn_leaky_relu_s8 ( q7_t *  in_out,
uint32_t  size,
q15_t  slope 
)

This function performs activation on signed 8-bit integer input vectors using the Leaky ReLU function, along with an in-place algorithm.

Parameters
[in,out]in_outPointer to the input/output vector
[in]sizeNumber of elements in the input/output vector
[in]slopeSlope value to be multiplied with the negative inputs. The result will be right shifted 15 bits to scale back to signed 8-bit integer.
Returns
None

Example:

#define SIZE 1024
q15_t slope = 16384;
q7_t in_out[SIZE] = {...};
riscv_nn_leaky_relu_s8(in_out, SIZE, slope);

This function performs activation on signed 8-bit integer input vectors using the Leaky ReLU function and applies asymmetric quantization to the outputs.

Parameters
[in]in_vecPointer to the input vector
[out]out_vecPointer to the output vector
[in]sizeNumber of elements in the input/output vector
[in]multi_identityScaling value for the quantization on the nonnegative inputs
[in]shift_identityShift amount for the quantization on the nonnegative inputs
[in]multi_alphaScaling value for the quantization on the negative inputs
[in]shift_alphaShift amount for the quantization on the negative inputs
[in]in_offsetOffset value for the input vector. It should be in the range of -128 to 127.
[in]out_offsetOffset value for the outputs. It should be in the range of -128 to 127.
[in]act_minMinimum value that the outputs are limited to. It should be in the range of -128 to 127.
[in]act_maxMaximum value that the outputs are limited to. It should be in the range of -128 to 127.
Returns
None
Note
During the quantization process, a positive shift_identity/shift_alpha value is used to left shift calculation results whereas a negative one is used to right shift.
